RetroAssembly v7 has been released!

This release brings several features and improvements. Here are the key highlights:

- Initial support for Nintendo 64
- Enhanced game media viewing experience
- Share your library with other users (selfhosted only)
- Enhanced virtual gamepad with multi-touch support
- Customizable gamepad hotkeys

## Dive into the new features

### Initial support for Nintendo 64

RetroAssembly now supports Nintendo 64 games by integrating the Mupen64Plus-Next core!

Limitations:

- It may crash at any time (like while launching, playing or saving) so do not rely on it too much.
- Analog key mapping is not yet supported. A controller is required to play most of the games.

![n64](https://github.com/user-attachments/assets/e170a7e4-384c-43f6-9b84-8e905fcad76b)

### Enhanced game media viewing

Game media has been completely overhauled with a new carousel interface, making it easier than ever to browse screenshots, box art, and other media for your games.

You can now upload any kind of file as game media, and we automatically detect the file type and display the appropriate preview. We now support images(jpg, png, etc), videos(mp4, mov, etc), and documents(pdf and cbz).

![media](https://github.com/user-attachments/assets/d65046dd-2a4d-48e8-a216-d0a031c1eb48)

### Enhanced virtual gamepad

The virtual gamepad has been improved for mobile and touch-screen users. We now make it work more like a physical gamepad, you can move your finger from one button to another without lifting your finger. Besides that, dedicated rewind and fast-forward buttons are now available.

![virtual-gamepad](https://github.com/user-attachments/assets/ae6a3cc9-ce09-4e49-b077-aef629931d90)

### Customizable gamepad hotkeys

Gamepad hotkeys for common actions can now be customized to your preference. You can select the button combinations for pause, rewind, and fast-forward.

![hotkeys](https://github.com/user-attachments/assets/b13c748a-676a-49f4-8a8d-9257aac8bace)

### Share your library with other users (self-hosted only)

For self-hosted instances with multiple users, we now allow you to share your library with other users, so it's nolonger needed to upload ROMs individually for each user.

![share](https://github.com/user-attachments/assets/7acfff3e-55b7-4435-985b-b3ceac09c676)
